javascript之this

2017-07-01
  • 原生 js

this关键字不同于其他编程语言地方在于,其指向并不是声明的位置,而是被调用时的执行环境,因此this的指向并不是固定不变的,而是动态的.

主要分为以下四种情况:

  • 作为普通函数调用
  • 作为对象的属性调用
  • new 对象
  • 通过call和apply方法显式绑定
  • 箭头函数

作为普通函数调用

当函数作为一个全局对象调用时,this指向全局对象,在浏览器环境中这个全局对象是window.

需要注意的是当处于严格模式时,this指向是undefined.

在实际应用,经常会遇到一些疑惑.比如DOM触发事件的回调函数中调用普通函数,this指向的是window或者undefined,而不是我们想要的节点.

作为对象的属性调用

当函数作为一个对象的属性调用时,this就指向了这个对象.
其实作为普通函数调用的方式可以理解作为window这个全局对象的属性(非严格模式下).

但是函数被赋值给另一个变量时,this指向可能会丢失,像作为普通函数一样:

new 对象

当java使用new关键字创建一个类时,我们常说通过构造函数实例化了一个对象.
在javascript中不同于此,使用new关键字并没有实例化一个对象.此处暂且不表.通过new关键字调用函数,this被指向了这个新创建的对象.

但是如果这个’构造函数’返回一个对象时,this的指向就变为返回的对象.

通过call和apply方法显式绑定

如上文所述,存在this指向全局对象window(或者undefined),以及指向丢失的问题.
既可如上文所示采用保存this引用的方法.也可以通过apply或者call方法将函数绑定到指定对象.
call和apply方法都是Function.prototype方法,在用法上第一个参数都是留给this,不同的是apply方法第二个参数接受一个数组或者类数组,作为参数传递给函数,用作参数个数未知的情况.如果参数个数已知可以使用call方法,一个个传入.

直接调用call或者apply方法,函数会立即执行,如果想要传递参数,可以创建一个包裹函数,传递进参数并返回函数

bind

上文所示的模式可以在很多流行的库中看到身影.在ES5中则直接提供了这种方法:Function.prototype.bind方法来简化.
bind方法一个应用情景是柯里化,预先为函数传递参数.这个时候如果没有使用this的话,可以传入null或者undefined.但是如果调用的函数内部使用this的话,this就绑定到全局对象中,这并不是我们想要的,可以传入一个特殊的空的对象ø = Object.create(null)消除副作用.

另一个场景是内部函数如setTimeout函数在回调函数中的this指向全局对象,因此需要通过bind将其显示的绑定到回调函数中,或者简单的保存this的引用.

箭头函数

在ES6中引入了箭头函数,不同于上述其他方法的是,箭头函数抛弃了this指向动态的说法,转而使用了词法作用域,也就是定义时所在的环境确定.确实可以减少初级错误.
